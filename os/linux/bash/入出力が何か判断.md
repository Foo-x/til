# 入出力が何か判断

- 標準入力
    - `[ -t 0 ]`
        - 端末
        - `read` を使うとユーザの入力を待つ
    - `[ -p /dev/stdin ]`
        - パイプ
        - 例: `echo foo | ./script.sh`
    - `[ -f /dev/stdin ]`
        - ファイル
        - 例: `./script.sh < foo.txt`
- 標準出力
    - `[ -t 1 ]`
        - 端末
        - コンソール上に表示される
    - `[ -p /dev/stdout ]`
        - パイプ
        - 例: `./script.sh | cat`
    - `[ -f /dev/stdout ]`
        - ファイル
        - 例: `./script.sh > foo.txt`
- 標準エラー出力
    - `[ -t 2 ]`
        - 端末
        - コンソール上に表示される
    - `[ -p /dev/stderr ]`
        - パイプ
        - 例: `./script.sh | cat`
    - `[ -f /dev/stderr ]`
        - ファイル
        - 例: `./script.sh 2> foo.txt`
